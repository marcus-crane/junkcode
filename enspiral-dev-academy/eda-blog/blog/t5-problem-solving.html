Can you tell me what .map() does?

Ooh, interesting. By the looks of it, .map() can be used to convert one array into another. For example, lets say you have the following array:

var numbers = [5, 10, 15, 20, 25]

If we wanted to multiply the contents by 4, we could do something overly complicated like looking that array then putting it back into another or, we could do this:

var fourTimes = numbers.map(function(num)) { 
                    return num * 4; 
                }

What we're doing is taking the numbers array and running .map on it. Our input for .map is a function which in this case, multiplies each number by 4. Our end result is something a little like this:

var fourTimes = [20, 40, 60, 80, 100]

The end result is a second array that has taken the original input and multipied the content by 4. Keep in mind that the original numbers array still exists alongside the new one.

.filter()?

The purpose of .filter() is, well, to filter, y'know? Let's say you have the following array

var numberArray = [2, 5, 9, 12, 14, 19]

If we wanted to filter out any number below 8, we can use .filter() like so

function underEight(number) {
                    return value >= 8;
                }

and then we can run our array through the function and we're done!

numberArray.filter(underEight);
                // numberArray is now [9, 12, 14, 19]

.reduce()?

Reduce, compress, same thing! The easiest version of .reduce() is using it to compress an array of numbers into a single value. So, you've got your array

var numbers = [0, 4, 8, 16]

But you want the total value of the numbers and looping each number, adding it to a value, is a bit too excessive. Here's where .reduce can come in handy

numbers.reduce(function(a, b) {
                    return a + b;
                });
                // total = 28
